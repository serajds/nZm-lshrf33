# تطبيق الموبايل — إدارة الإشراف والمتابعة

تطبيق Android كامل الميزات مبني بـ Expo، يستخدم نفس الواجهة الخلفية للتطبيق الويب
(`@workspace/api-server`). الواجهة عربية بالكامل واتجاه RTL.

## الشاشات والميزات

- **تسجيل الدخول** — برقم الهاتف وكلمة المرور.
- **الرئيسية** — لوحة معلومات سريعة + قائمة بأحدث المشاريع.
- **المشاريع** — قائمة كاملة + بحث + شاشة تفاصيل لكل مشروع.
- **التقارير** — عرض/إنشاء/اعتماد/إرجاع/حذف (حسب الصلاحيات).
- **الأنشطة** — عرض الأنشطة مجمَّعة بحسب المجموعة مع شريط تقدم.
- **النماذج** — تعبئة وإرسال نماذج المشاريع المخصَّصة.
- **الحضور** — تسجيل حضور/انصراف بصورة سيلفي + موقع جغرافي + **طابور غير متصل**:
  إذا انقطع الإنترنت يحفظ العملية محلياً ويرسلها تلقائياً عند عودة الاتصال
  (مفتاح الـ idempotency `clientId` يمنع التكرار).
- **الإدارة** (للمسؤولين) — قوائم المستخدمين والشركات وسجل الأحداث.
- **إشعارات Push عبر Expo** — يصل تنبيه فوري لمنشئ التقرير عند اعتماد تقريره.

---

## التشغيل أثناء التطوير

التطبيق يعمل تلقائيًا عبر workflow اسمه `artifacts/mobile: expo`. تطبيق
**Expo Go** على هاتفك يعمل مباشرة بمسح رمز QR من شريط عنوان Replit.

```
EXPO_PUBLIC_DOMAIN  →  يُحقن من workflow تلقائيًا = REPLIT_DEV_DOMAIN
```

كل طلبات API تذهب إلى نفس النطاق (`/api/...`) لذا لا يلزم تكوين إضافي للتجربة المحلية.

> **ملاحظة عن إشعارات Push**: لا تعمل في **Expo Go** على Android (منذ SDK 53)
> — يجب بناء **APK** عبر EAS Build (الخطوات أدناه) لاختبارها.

## بناء ملف APK لتجربته على هواتف Android (EAS Preview)

نستخدم خدمة **EAS Build** السحابية المجانية من Expo. الخطوات تتم على جهازك المحلي
(لا يمكن تشغيل `eas build` داخل بيئة Replit).

### 1. تجهيز حساب Expo (مرة واحدة)

1. أنشئ حسابًا مجانيًا على <https://expo.dev/signup>.
2. على جهازك المحلي ثبّت EAS CLI:
   ```bash
   npm install -g eas-cli
   eas login
   ```

### 2. ربط المشروع بـ EAS (مرة واحدة)

من داخل مجلد التطبيق:

```bash
cd artifacts/mobile
eas init
```

سيُحدِّث الأمر ملف `app.json` بحقل `extra.eas.projectId`. **هذا الحقل ضروري
لإشعارات Expo Push** — بدونه لن يتم تسجيل جهازك لدى خدمة الإشعارات.

### 3. تعديل `eas.json`

افتح `artifacts/mobile/eas.json` واستبدل القيمة `REPLACE_WITH_YOUR_REPLIT_APP_DOMAIN`
في كلا البروفايلين (`preview` و `production`) بنطاق تطبيقك المنشور على Replit.

مثال:

```json
"env": {
  "EXPO_PUBLIC_DOMAIN": "construction-supervision-yourname.replit.app"
}
```

> النطاق هو الذي يظهر بعد نشر التطبيق الويب باستخدام **Publish** في Replit.
> بدون هذه القيمة، لن يعرف تطبيق الموبايل المُجمَّع كيف يصل إلى الـ API.

### 4. بناء APK للتجربة

```bash
cd artifacts/mobile
eas build -p android --profile preview
```

سيستغرق البناء ~10–20 دقيقة على خوادم Expo. عند الانتهاء، ستحصل على رابط لتنزيل
ملف `.apk` يمكن تثبيته مباشرة على أي هاتف Android (دون الحاجة إلى Google Play).

### 5. (لاحقًا) النشر على Google Play

بعد التجربة، استخدم بروفايل `production` الذي يُنتج ملف `.aab` صالحًا للرفع على
المتجر:

```bash
eas build -p android --profile production
```

## الأذونات المطلوبة

عند أول استخدام، سيطلب التطبيق:

- **الكاميرا** — لالتقاط صورة سيلفي عند تسجيل الحضور.
- **الموقع الجغرافي** — للتحقق من تواجد الموظف داخل نطاق المشروع.
- **الإشعارات** — لاستقبال تنبيهات اعتماد التقارير.

> يقوم الخادم بالتحقق من المسافة (geofence) مقارنةً بإحداثيات الموقع. إن كان
> المستخدم خارج النطاق سيرفض الخادم العملية ويُعرض له تنبيه باللغة العربية.

## إشعارات Expo Push — كيف تعمل؟

1. بعد تسجيل الدخول، يطلب التطبيق إذن الإشعارات ويحصل على `ExponentPushToken`.
2. يُرسَل التوكن إلى الخادم عبر `POST /api/push/expo/register` ويُحفَظ في
   جدول `expo_push_tokens` مرتبطًا بالمستخدم.
3. عند اعتماد تقرير (`PATCH /api/projects/:projectId/reports/:id/status`)
   يستدعي الخادم `sendExpoPushToUser` لإرسال إشعار لمنشئ التقرير عبر
   `https://exp.host/--/api/v2/push/send`.
4. عند الضغط على الإشعار، ينقل التطبيق المستخدم مباشرةً إلى صفحة التقرير.
5. التوكنات المعطَّلة (`DeviceNotRegistered`) تُحذف تلقائياً من قاعدة البيانات.

> ملاحظة: التطبيق يستخدم **خدمة Expo المجانية** ولا يحتاج إلى مفتاح FCM خاص.

## ملاحظات تقنية

- **التخزين الآمن**: التوكن يُحفظ عبر `expo-secure-store` (Keystore على Android).
- **Idempotency**: كل طلب حضور/انصراف يحمل `clientId` (UUID v4) لمنع التكرار،
  حتى عند إعادة المحاولة من طابور الـ offline.
- **حجم الصورة**: تُلتقط الصورة بجودة 0.7 ثم يقوم العميل بتقليل الأبعاد إلى
  1024px وضغطها إلى JPEG 0.75 قبل الإرسال.
- **اتجاه الواجهة**: يُستدعى `I18nManager.forceRTL(true)` عند التشغيل الأول.
- **الأدوار المحظورة من الحضور**: `owner` و `contractor` (نفس قاعدة الواجهة الخلفية).
- **Cache بيانات الـ React Query**: يتم تحديث القوائم تلقائياً عند سحب الشاشة لأسفل.

## Forbidden

- لا تستخدم `npx expo start` مباشرة في Replit — استخدم زر إعادة تشغيل الـ workflow.
- لا تُغيّر `bundleIdentifier` أو `package` بعد أول بناء (يكسر التحديثات اللاحقة).
- لا تُجرِ `eas build` داخل Replit — فقط على جهازك المحلي.
